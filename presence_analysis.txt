ANÁLISE DO SISTEMA DE PRESENÇA ONLINE ATUAL
=============================================

COMO FUNCIONA HOJE:
1. profiles.online_status = 1 (online) ou 2 (offline)
2. profiles.last_seen_at = timestamp
3. auth_provider.dart seta online_status=1 no login e online_status=2 no logout
4. onlineMembersCountProvider faz query: SELECT profiles WHERE online_status=1
5. _buildOnlinePage filtra membros onde profiles.online_status == 1
6. community_members_screen.dart mostra indicador verde se online_status == 1

PROBLEMAS:
1. Se o app crashar ou o usuário fechar sem logout, online_status fica 1 para sempre
2. Não há heartbeat - se perder conexão, continua "online"
3. É uma query pontual (FutureProvider), não atualiza em tempo real
4. Não usa Supabase Realtime Presence (que é feito para isso)

SOLUÇÃO:
1. Criar PresenceService que usa Supabase Realtime Presence channels
2. Cada comunidade tem um channel: "presence:community:{id}"
3. Heartbeat automático via Presence (Supabase gerencia automaticamente)
4. Provider StreamProvider que escuta mudanças de presença em tempo real
5. Atualizar profiles.online_status e last_seen_at via trigger/cron no backend
6. Fallback: se Presence não disponível, usar heartbeat manual com last_seen_at

ARQUIVOS A MODIFICAR:
- CRIAR: core/services/presence_service.dart
- CRIAR: core/providers/presence_provider.dart
- MODIFICAR: community_detail_screen.dart (onlineMembersCountProvider → presença real)
- MODIFICAR: community_members_screen.dart (isOnline → presença real)
- MODIFICAR: auth_provider.dart (iniciar/parar presença)
- MODIFICAR: main.dart (inicializar presença)
